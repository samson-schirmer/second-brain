# Writer Agent Deployment

**What:** Building and deploying Writer AI agents across ~12+ marketing teams at Comcast.

**Status:** In progress. 21+ agents currently deployed; targeting 38+ with validated ROI. Current expansion areas include multicultural translation, presentation-builder playbooks, Comcast Business/GTM brief generation, and connector-enabled Microsoft workflows.

**Priority:** Highest.

**Owner:** Samson Schirmer.

**Key collaborators:** [[marcel]] (building and demoing), [[chennai-team]] (onboarding), [[abbey]], marketing team stakeholders.

**Done signal:** 38+ agents deployed, validated ROI numbers (not estimates), business transformation story packaged and traveling at SVP level without Samson pushing it. Teams pulling — requesting agents because they've seen results elsewhere.

**Validates:** The [[scot-engle]] ask. ROI numbers must be stamped by [[seth-monahan]]'s team before going public.

**Lifecycle:** All agents follow the [[adlc]] — Intake → Design → Build → Test → Deploy → Monitor → Govern.

**Key insight:** Agents reach "functionally complete" quickly, but "reliable and trustworthy" takes continuous iteration. This distinction shapes timelines and resource expectations.

## Current build patterns from team coordination transcript

- Presentation builders need discovery questions and story-shaping steps before generating slides.
- Translation agents need controlled stakeholder feedback loops and brand-owner governance before terminology is reflected in Brand Hub or Writer.
- Connector-enabled agents can use Outlook, Teams, SharePoint, OneDrive, and Calendar once users request MCP server access through My Comcast Access.
- Repetitive workstreams are the best candidates for deeper playbook investment because the same meeting or deliverable repeats weekly, monthly, or quarterly.
